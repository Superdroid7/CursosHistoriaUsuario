Registro de Errores y Soluciones: Bloque 1

Documentación de las dificultades técnicas enfrentadas durante el desarrollo de la Base Hexagonal y Vista Pública.

Dificultad 1: Error de Importación de Swagger

IDE / Gradle

El problema: Al escribir las anotaciones de documentación en los controladores (@Tag, @Operation), el IDE (Visual Studio Code/Eclipse) marcaba la línea en rojo con el error: The import io.swagger cannot be resolved.

La solución: El error se debió a un desajuste de sincronización. Aunque habíamos agregado la librería correcta (springdoc-openapi-starter-webmvc-ui) en el archivo build.gradle, el editor de código no se había enterado de la actualización y no había descargado los archivos JAR de internet. Se solucionó ejecutando el comando del editor para limpiar y recargar el proyecto Gradle (Java: Clean workspace o recargando la pestaña de Gradle).

Dificultad 2: Escaneo de Entidades en Arquitectura Hexagonal

Arquitectura Spring

El problema: Spring Boot, por defecto, solo busca entidades (@Entity) y repositorios en la misma carpeta raíz o por debajo de donde está la clase IaScursosApplication.java. Debido a nuestra arquitectura de carpetas separadas (domain, infrastructure, application), la base de datos no lograba construir las tablas automáticamente porque "no las veía".

La solución: Tuvimos que indicarle explícitamente a Spring dónde buscar estas clases. Añadimos las anotaciones @EntityScan(basePackages = "...") y @EnableJpaRepositories(basePackages = "...") en la clase principal del proyecto, dándole las rutas exactas de la capa de infraestructura. Con esto, Spring mapeó correctamente la base de datos.
