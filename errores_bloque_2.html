Registro de Errores y Soluciones: Bloque 2

Documentación de las dificultades técnicas enfrentadas durante el desarrollo de la Seguridad JWT.

Dificultad 1: Falta de Contexto para Iniciar Sesión en Swagger

Experiencia de Usuario

El problema: Una vez activamos la seguridad de Spring, todos los endpoints requerían un Token JWT. Sin embargo, no teníamos credenciales (correo/contraseña) registradas en la base de datos para ingresar, dejando el sistema hermético. El usuario de Swagger no sabía qué datos poner.

La solución: Se creó la clase DataInitializer.java, que implementa CommandLineRunner. Este componente actúa como una "semilla" (seed). Al arrancar el servidor, verifica si existe un administrador; si no lo hay, crea automáticamente el usuario maestro admin@ias.com con la contraseña admin123. Esto resolvió el bloqueo y permitió la generación del Token.

Dificultad 2: Configuración del Secreto JWT en el archivo YML

Seguridad / Configuración

El problema: La firma de los Tokens JWT necesita una "Clave Secreta" muy larga. Colocar esta clave quemada directamente en el código Java es una mala práctica de seguridad que vulnera el sistema si el código fuente es robado.

La solución: El desconocimiento inicial sobre el archivo application.yml se resolvió con documentación. Se movió la clave secreta y el tiempo de expiración (24h) a este archivo YAML utilizando las variables jwt.secret y jwt.expiration. De este modo, en producción, la clave se reemplazará por verdaderas variables de entorno del servidor, manteniendo el código Java limpio y blindado.
