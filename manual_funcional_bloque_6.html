Manual Funcional: Validaciones de Usuario

Hemos activado escudos protectores. Ya no puedes guardar cursos con información defectuosa.

Guía de Pruebas en Swagger

Paso 1: Provocar el escudo de validación (HU-014)
1. Inicia sesión en Swagger con tu token.
2. Intenta crear un Curso nuevo, pero en el campo courseUrl escribe simplemente "www.google.com" (sin el https).
3. Presiona Execute.
Resultado esperado: El sistema rechazará la petición y verás un JSON formateado de la siguiente manera:
{
  "message": "Error de validación en los campos enviados",
  "data": {
    "courseUrl": "El formato de la URL es incorrecto, debe incluir http o https"
  }
}
Paso 2: Comprobar el mensaje de éxito (HU-015)
1. Ahora corrige el JSON y pon una URL válida como "https://www.google.com".
2. Ejecuta nuevamente.
Resultado esperado: En lugar de recibir solo los datos del curso, ahora recibirás el mensaje visual empaquetado confirmando tu acción:
{
  "message": "Curso creado exitosamente",
  "data": {
    "id": 5,
    "title": "Curso...",
    ...
  }
}
