Anatomía del application.yml

A continuación, desglosaremos línea por línea el "Menú de Configuraciones" de tu aplicación. Presta mucha atención a la sintaxis ${VARIABLE:VALOR_POR_DEFECTO}. Esta sintaxis mágica le dice a la aplicación: "Busca esta VARIABLE en el entorno (ej. en tu servidor de la nube). Si no la encuentras, usa el VALOR_POR_DEFECTO que está después de los dos puntos."

1. Configuración Principal y Base de Datos

spring:
  application:
    name: IAScursos
Agrupa todas las configuraciones nativas de Spring Boot. Aquí simplemente le estamos dando un nombre interno a nuestra aplicación ("IAScursos").
  datasource:
    url: ${SPRING_DATASOURCE_URL:jdbc:postgresql://localhost:5432/iascursos}
    username: ${SPRING_DATASOURCE_USERNAME:postgres}
    password: ${SPRING_DATASOURCE_PASSWORD:postgres}
    driver-class-name: org.postgresql.Driver
Esta es la brújula para encontrar tu Base de Datos PostgreSQL. Usa las variables mágicas. Por ejemplo, si en la nube configuras SPRING_DATASOURCE_USERNAME como "admin", usará "admin". Si estás en tu PC y no existe esa variable, usará el valor por defecto: postgres. El driver-class-name es el motor interno que Spring usa para hablar el idioma de PostgreSQL.

2. Configuración de JPA e Hibernate (El ORM)

  jpa:
    hibernate:
      ddl-auto: update
ddl-auto: update es una instrucción clave. Le dice a Hibernate (la herramienta que traduce tus clases de Java a tablas de base de datos): "Revisa mis clases Java. Si hay una clase nueva (como Admin) y no existe la tabla en PostgreSQL, créala automáticamente. Si añadí una columna, actualiza la tabla."
    properties:
      hibernate:
        dialect: org.hibernate.dialect.PostgreSQLDialect
        format_sql: true
    show-sql: true
El dialect le dice a Hibernate qué versión/dialecto exacto de SQL debe escribir (en este caso PostgreSQL). show-sql y format_sql le ordenan a la aplicación que imprima en tu consola todas las consultas (SELECT, INSERT, UPDATE) que hace a la base de datos en tiempo real, formateadas de forma bonita para que puedas leerlas mientras el programa corre.

3. Documentación (Swagger)

springdoc:
  swagger-ui:
    path: /swagger-ui.html
    operationsSorter: method
  api-docs:
    path: /v3/api-docs
Configura la herramienta SpringDoc. swagger-ui.path define la URL exacta que pones en tu navegador para ver la interfaz visual. operationsSorter: method hace que los endpoints en la página se ordenen alfabéticamente por su método HTTP (DELETE, GET, POST, PUT). api-docs.path es la ruta interna de la que Swagger saca la información en formato JSON crudo para armar la página visual.

4. Seguridad (JSON Web Tokens)

jwt:
  secret: ${JWT_SECRET:404E6352...}
  expiration: ${JWT_EXPIRATION:86400000}
Estos son valores personalizados que nosotros creamos (no son nativos de Spring, sino creados por nosotros para nuestro JwtService). secret es la llave maestra ultra secreta (un código hexadecimal muy largo) que se usa para firmar y verificar si un Token es auténtico o fue falsificado. expiration es el tiempo de vida del Token en milisegundos (86400000 milisegundos equivalen exactamente a 24 horas). Al igual que la base de datos, si los configuras en la nube usando variables de entorno, sobreescribirán los valores por defecto.
