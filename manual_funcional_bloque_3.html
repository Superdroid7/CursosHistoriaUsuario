Manual Funcional: Gestionando Cursos

¡El sistema de administración ya tiene poder de creación y destrucción!

¿Qué hace el sistema ahora?

  • Crear y Editar: El administrador puede agregar nuevos cursos con una URL de imagen, un enlace, y un título.
  • Borrado Lógico: Si te equivocas o un curso ya no sirve, puedes borrarlo. El curso no se borrará para siempre de la base de datos, sino que se "ocultará" de la vista pública para preservar el historial.
  • Validaciones: El sistema ahora te lanzará un error 400 Bad Request si intentas guardar un curso sin nombre o sin categoría.

Guía de Pruebas en Swagger

Paso 0: Preparativos
1. Inicia sesión en POST /api/auth/login con admin@ias.com y admin123 para obtener tu Token.
2. Sube al tope de la página de Swagger, presiona el botón verde "Authorize", pega tu token y dale a "Authorize". ¡Ahora tienes permisos!
Paso 1: Crear un Curso
Atención: Para crear un curso, necesitas asignarlo a una categoría existente en la base de datos. Como en este momento tu tabla de categorías está vacía, no podrás guardar cursos exitosamente (el sistema te protegerá y lanzará un error). En el Bloque 4 implementaremos la creación de categorías.
Paso 2: Rutas Privadas vs Públicas
Si crearas un curso exitosamente, y luego usas DELETE /api/admin/courses/1, el curso será desactivado. Al buscar en la lista pública GET /api/public/courses no aparecerá, pero si usas el listado histórico de admin GET /api/admin/courses, ¡ahí seguirá visible!

El próximo paso (Bloque 4) es darle vida al CRUD de Categorías, para que podamos empezar a popular nuestro catálogo real y unirlo a los cursos. ¡Vamos genial!
