Registro de Errores y Soluciones: Bloque 3

Documentación de las dificultades técnicas enfrentadas durante el desarrollo del CRUD de Cursos.

Dificultad 1: Error de Importación io.jsonwebtoken

IDE / Dependencias

El problema: Al crear la clase JwtService.java, el IDE arrojó líneas rojas con el mensaje: The import io.jsonwebtoken cannot be resolved, imposibilitando compilar la creación de los tokens JWT.

La solución: Similar al incidente del Bloque 1, fue un problema de desincronización. Habíamos agregado exitosamente jjwt-api, jjwt-impl y jjwt-jackson al build.gradle. Se resolvió forzando al editor a releer el archivo Gradle para descargar las librerías a la caché local de Java.

Dificultad 2: Gestión de Archivos para la Decisión del Tipo de Borrado

Diseño Arquitectónico

El problema: En el plan de arquitectura surgió la pregunta de si los cursos debían eliminarse de raíz (Hard Delete) o solo ocultarse para guardar un historial de auditoría (Soft Delete). Se eligió el Soft Delete, pero se generó la necesidad de guardar el código del Hard Delete "por si acaso", sin que esto ensucie el código principal ni se suba a GitHub.

La solución: Se implementó el Soft Delete añadiendo el campo isActive en la base de datos y creando filtros personalizados en JpaCourseRepository. Para resolver el almacenamiento del código descartado, se creó un archivo "Scratch" (borrador temporal) llamado hard_delete_backup.java en un directorio local ignorado por Git, conteniendo el algoritmo nativo de borrado por ID.
