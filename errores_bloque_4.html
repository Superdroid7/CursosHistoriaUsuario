Registro de Errores y Soluciones: Bloque 4

Documentación de las dificultades técnicas enfrentadas durante el desarrollo del CRUD de Categorías.

Dificultad 1: Creación de Paquete Incorrecto (Dedazo)

CLI / Organización

El problema: Apareció el error de compilación: The declared package "com.crusos...in.web.dto" does not match the expected package "" en el archivo CategoryRequest.java. Esto se debió a un error tipográfico humano: al momento de crear la estructura de carpetas, se utilizó un punto (in.web) en lugar de una barra diagonal (in/web), lo que provocó que Java interpretara mal la ubicación del paquete.

La solución: Se solucionó abriendo la terminal de comandos (Bash/Powershell) y ejecutando el comando mv para mover el archivo CategoryRequest.java a la ruta correcta con barras (src/main/java/com/.../in/web/dto/). Posteriormente, se eliminó la carpeta corrupta con rm -r. El IDE reconoció de inmediato el archivo en el paquete válido y el error desapareció.

Dificultad 2: Riesgo de Corrupción de Datos (Orfandad)

Base de Datos / Diseño

El problema: Al diseñar el borrado de Categorías, nos enfrentamos a la HU-013. Si un administrador borra la categoría "Programación", ¿qué pasa con los 50 cursos que estaban bajo esa categoría? Podrían ocurrir fallos en cascada o la pantalla de visitantes podría romperse al intentar cargar un curso con una categoría inexistente.

La solución: En lugar de permitir el borrado o hacer un borrado automático de todos los cursos en cascada (lo cual es muy destructivo), se optó por un Bloqueo de Integridad. En AdminCategoryUseCase.java, antes de ejecutar deleteById, el sistema busca cursos asociados. Si la lista no está vacía, arroja un IllegalStateException. El controlador captura este error y devuelve un código 409 Conflict, obligando al administrador a vaciar la categoría de forma manual y consciente antes de poder borrarla.
