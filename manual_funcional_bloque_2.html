Manual Funcional: Novedades del Bloque 2

¡El sistema ya tiene su propia aduana de seguridad funcional!

¿Qué hace el sistema ahora?

  • Público vs Privado: Todo lo que hicimos en el Bloque 1 (Cursos, Categorías, y la vista de Swagger) ya no pide contraseña. Cualquier usuario visitante puede entrar libremente.
  • Autenticación Real (JWT): Ahora existe un proceso de Login de verdad para administradores. La aplicación te otorga un "Token" que actúa como un pase VIP.
  • Administrador por Defecto: El sistema ahora se autoconfigura creando un administrador cuando lo arrancas. Las credenciales son:

    Email: admin@ias.com
    Contraseña: admin123

¿Cómo probarlo en Swagger?

Paso 1: Entra a http://localhost:8080/swagger-ui.html. ¡Verás que ya no salta la molesta ventana blanca pidiendo contraseña predeterminada!
Paso 2: Busca la nueva sección Authentication y abre el endpoint verde POST /api/auth/login.
Paso 3: Dale click a "Try it out". En el cuadro de texto del Request Body, pon el email y password correctos:

{
  "email": "admin@ias.com",
  "password": "admin123"
}
Paso 4: Presiona "Execute". Verás que el servidor te responde con un código 200 OK y un texto largo (que parece una mezcla aleatoria de letras y números): ¡Ese es tu Token JWT! Es tu pase VIP que dura 24 horas.

En el próximo bloque de historias (CRUD de cursos), usaremos ese código (Token) para darle permiso a Swagger de crear y borrar contenido. ¡Avanzando firme hacia el Bloque 3!
