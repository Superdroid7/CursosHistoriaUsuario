Manual Funcional y Contexto General (Bloque 1)

Bienvenido al Manual Funcional. A diferencia de la documentación técnica (que explica el código), este documento te explica qué hace tu sistema hoy, cómo interactuar con él como usuario y por qué se comporta de ciertas maneras.

1. ¿Qué sabe hacer el sistema actualmente?

Hemos completado la Épica 1: Visualización pública de cursos. A nivel de funcionalidades reales, tu sistema ahora mismo cuenta con:

  • Una base de datos conectada: Las tablas virtuales para Cursos y Categorías ya existen, aunque actualmente están vacías.
  • Endpoint Público de Cursos: Si le pides al sistema "dame todos los cursos" (usando la ruta /api/public/courses), el sistema irá a la base de datos y te devolverá la lista.
  • Filtro de Cursos: Puedes pedirle al sistema "dame los cursos de la categoría X" y sabrá filtrarlos.
  • Endpoint Público de Categorías: Puedes pedirle la lista de categorías disponibles para alimentar el filtro de tu Landing Page en el futuro (ruta /api/public/categories).

2. El "Problema" de Swagger: ¿Por qué me pide contraseña?

El Sistema de Defensa de Spring Security (El Muro de Demacia)
En tu archivo de configuración (build.gradle), tienes instalada una dependencia llamada spring-boot-starter-security. Esta herramienta es como un guardia de seguridad paranoico que, por defecto, bloquea absolutamente todo el sistema (incluso la documentación de Swagger) hasta que le enseñes una identificación.

Como aún no hemos llegado a la Épica 2 (donde programaremos el login de Administrador de verdad con contraseñas reales y bases de datos), Spring Boot genera una contraseña aleatoria de emergencia cada vez que arranca la aplicación.

¿Cómo entro a Swagger ahora mismo?

Paso 1: Ve a tu consola (terminal de PowerShell donde corriste .\gradlew bootRun).
Paso 2: Busca entre los textos blancos de inicio una línea que dice algo como:
Using generated security password: 6a8d7e9f-1234-abcd-5678-90abcdef1234
Paso 3: Ve a Swagger (http://localhost:8080/swagger-ui.html). Cuando te pida credenciales, ingresa:
  • Username: user
  • Password: Pega la contraseña generada que encontraste en la consola.
Nota sobre el Futuro (Bloque 2):
En el próximo bloque vamos a "educar" a este guardia de seguridad. Le enseñaremos que la ruta de Swagger y las rutas /api/public/... deben estar abiertas para todos sin pedir contraseña. Y crearemos el sistema de Login real donde tú definirás tu propio usuario y clave de administrador.

3. ¿Cómo probar lo que hemos hecho?

Una vez que logres entrar a Swagger, verás dos pestañas (Endpoints):

  • GET /api/public/courses: Al expandirlo y presionar "Try it out" y luego "Execute", verás que el sistema te responde con un código 200 OK y un arreglo vacío []. Esto es correcto, ¡porque el sistema funciona, pero aún no has creado cursos!
  • GET /api/public/categories: Hará lo mismo, devolviendo un arreglo vacío de categorías.

Este manual te servirá de brújula a medida que el sistema crezca. ¡Estamos listos para el Bloque 2!
