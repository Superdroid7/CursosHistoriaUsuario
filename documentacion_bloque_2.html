Reporte de Misión: Bloque 2 - Seguridad JWT y Filtros

En este bloque instalamos el candado principal del sistema. Nadie entra a los controladores de administrador sin un pase válido.

1. Filtros de Seguridad (JwtAuthenticationFilter.java)

[Satisface HU-004] Inicio de sesión protegido y [Satisface HU-005] Validación de Lista Negra

Este archivo es un vigilante que revisa todas las peticiones entrantes y valida que el token no haya sido anulado (Logout).

        jwt = authHeader.substring(7);

        // HU-005: Validar lista negra
        if (revokedTokenRepository.findByToken(jwt).isPresent()) {
            response.setStatus(HttpServletResponse.SC_UNAUTHORIZED);
            return;
        }
  • OncePerRequestFilter: Intercepta la petición web buscando la cabecera (header) donde está escondido el Token JWT.
  • Validación de Lista Negra (Logout): Consultamos a la base de datos si el token actual está en la tabla `revoked_tokens`. Si es así, cortamos la conexión de inmediato enviando un SC_UNAUTHORIZED (401) porque el usuario cerró sesión.
Analogía de Silent Hill 2: Imagina que el Controlador es la habitación secreta en el hotel. El JwtAuthenticationFilter es Pyramid Head custodiando el pasillo. Cada vez que alguien intenta pasar, le revisa los bolsillos buscando el Token. Pero además, Pyramid Head tiene un cuaderno negro (Lista Negra / RevokedTokenRepository). Si tú vienes con una llave, pero el cuaderno dice que esa llave está anulada (porque cerraste sesión), Pyramid Head te rechaza el paso inmediatamente (401).
