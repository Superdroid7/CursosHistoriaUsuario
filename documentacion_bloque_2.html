Reporte de Misión: Bloque 2 - Seguridad JWT y Filtros

En este bloque instalamos el candado principal del sistema. Nadie entra a los controladores de administrador sin un pase válido.

1. Filtros de Seguridad (JwtAuthenticationFilter.java)

Este archivo es un vigilante que revisa todas las peticiones entrantes.

@Component
public class JwtAuthenticationFilter extends OncePerRequestFilter {
    @Override
    protected void doFilterInternal(HttpServletRequest request, HttpServletResponse response, FilterChain filterChain) {
        ...
        filterChain.doFilter(request, response);
    }
}
  • OncePerRequestFilter: Es una clase abstracta de Spring que garantiza que este filtro se ejecutará exactamente una vez por cada petición web (para no hacer validaciones dobles y desperdiciar rendimiento).
  • HttpServletRequest: Contiene toda la información que envió el usuario: su IP, la URL que intenta visitar, y lo más importante: la "cabecera" (header) donde está escondido el Token JWT.
  • FilterChain: Es el "pase de siga". Si el token es válido, invocamos filterChain.doFilter para que la petición siga su camino hacia el Controlador. Si no lo llamamos, la petición muere ahí mismo (Acceso denegado).
Analogía de Silent Hill 2: Imagina que el Controlador (el destino final) es la habitación secreta en el hotel. El JwtAuthenticationFilter es Pyramid Head custodiando el pasillo. Cada vez que alguien (una petición web) intenta pasar, Pyramid Head lo detiene y le revisa los bolsillos buscando una carta específica (El Token JWT). Si la carta es válida, Pyramid Head se hace a un lado (filterChain.doFilter) y le deja pasar. Si no la tiene o es falsa, lo ejecuta inmediatamente impidiendo que avance.

2. Fabricación del Token JWT (JwtService.java)

Este archivo es el que "imprime" los pases de abordar (Tokens) cuando el admin pone su usuario y contraseña correctamente.

String token = Jwts.builder()
    .setSubject(user.getEmail())
    .setIssuedAt(new Date(System.currentTimeMillis()))
    .setExpiration(new Date(System.currentTimeMillis() + expiration))
    .signWith(getSigningKey(), SignatureAlgorithm.HS256)
    .compact();
  • setSubject: Define "de quién" es este token. Usamos el correo electrónico (email) del administrador.
  • setIssuedAt / setExpiration: Definen la fecha de expedición y caducidad. Si un hacker roba el token, solo le servirá por 24 horas antes de autodestruirse lógicamente.
  • signWith: Cifra matemáticamente el token usando la Clave Secreta del servidor (inyectada desde application.yml). Si alguien altera una sola letra del token, la firma HS256 se rompe y el sistema detectará que es un token falso.

3. Semilla de Arranque (DataInitializer.java)

@Component
public class DataInitializer implements CommandLineRunner {
    @Override
    public void run(String... args) { ... }
}
  • CommandLineRunner: Es una interfaz especial de Spring Boot. Le indica al sistema que, en el instante exacto en que el servidor termina de arrancar, debe ejecutar el código dentro del método run. Lo usamos para crear al administrador admin@ias.com si la base de datos está vacía.
