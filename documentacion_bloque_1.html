Reporte de Misión: Bloque 1 - Base Hexagonal y Vista Pública

En esta etapa fundacional, construimos la estructura base del backend e implementamos el listado de cursos para el público.

1. Arquitectura Hexagonal (Ports & Adapters)

Analogía de League of Legends: Imagina que tu código central (Reglas de negocio / Dominio) es el Nexo. El Nexo no ataca ni interactúa directamente con los campeones enemigos; depende de las Torretas y los Súbditos (Adaptadores) para interactuar con el mundo exterior. Los "Puertos" son las ranuras donde conectas esas defensas. Si mañana cambias de base de datos (pasas de PostgreSQL a MySQL), tu Nexo queda intacto, solo cambias la Torreta. Todo el núcleo del juego sigue funcionando igual.

2. Explicación de Código: Entidades vs Modelos

En el archivo CourseEntity.java configuramos cómo se guarda un curso en la base de datos.

@Entity
@Table(name = "courses")
public class CourseEntity {
    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;
    ...
}
  • @Entity: Esta anotación de Spring le dice al sistema: "Esta clase no es código normal, es la representación exacta de una tabla en la base de datos".
  • @Table(name = "courses"): Fuerza a que la tabla en PostgreSQL se llame "courses" en minúsculas y plural, siguiendo las buenas prácticas.
  • @Id y @GeneratedValue: Indican que el campo id es la llave primaria (Primary Key) y que la base de datos debe autoincrementar el número (1, 2, 3...) por cada curso nuevo que se inserte.
Analogía de Resident Evil: El Course.java (Dominio) es el concepto de una Planta Verde en la mente del personaje. El CourseEntity.java es cómo guardas esa Planta Verde exactamente en las celdas cuadradas del Maletín de inventario (Base de datos). Las anotaciones `@Entity` son las reglas físicas de cuántos bloques del maletín ocupa.

3. Explicación de Código: El Controlador Público

En PublicCourseController.java abrimos la puerta para que las páginas web consuman los cursos.

@RestController
@RequestMapping("/api/public/courses")
public class PublicCourseController {
    @GetMapping
    public ResponseEntity<List<Course>> getAllCourses() { ... }
}
  • @RestController: Indica que esta clase recibe peticiones de internet y responde con datos puros (generalmente en formato JSON), no con páginas web enteras.
  • @RequestMapping: Define la "dirección web" base. Cualquier petición que llegue a /api/public/courses será atrapada por este archivo.
  • @GetMapping: Especifica que este método solo se activa si el usuario "pide" (GET) datos. Si intentara enviar datos (POST) o borrarlos (DELETE), este método lo ignoraría.
  • ResponseEntity: Es una clase envoltorio (wrapper). No solo devuelve la lista de cursos, sino que le adjunta un código de estado HTTP (ej. 200 OK), indicando al navegador que la operación fue un éxito total.
