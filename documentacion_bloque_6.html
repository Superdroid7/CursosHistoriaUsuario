Reporte de Misión: Bloque 6 - Experiencia y Validaciones

Hemos finalizado la Épica 6 y blindado el sistema para que ningún administrador pueda corromper la base de datos por errores de escritura, y para que el frontend siempre reciba mensajes claros.

1. Captura Global de Errores (@ControllerAdvice)

En lugar de poner un try-catch en cada uno de nuestros controladores (lo cual ensuciaría el código), usamos una clase mágica llamada GlobalExceptionHandler.java.

@ControllerAdvice(basePackages = "com.crusos.infrastructure.adapter.in.web")
public class GlobalExceptionHandler {
    @ExceptionHandler(MethodArgumentNotValidException.class)
    public ResponseEntity<ApiResponse<Map<String, String>>> handleValidationExceptions(...) { ... }
}
  • @ControllerAdvice: Es un interceptor global. Se sienta "por encima" de todos los controladores de la carpeta especificada. Su único trabajo es vigilar si alguno explota.
  • @ExceptionHandler: Le dice a Spring: "Si cualquier parte del programa lanza la excepción específica MethodArgumentNotValidException (que es la que se lanza cuando fallan las validaciones @Valid o @URL), atráparla aquí mismo antes de que rompa el servidor, y ejecuta este bloque de código para responder de forma bonita".
Analogía de League of Legends: Imagina que @ControllerAdvice es el ítem Velo del Hada de la Muerte (Banshee's Veil). En lugar de tener que esquivar activamente cada hechizo enemigo (que sería poner un try-catch manual en cada función), este ítem pasivo te cubre como una burbuja. Si una petición maligna o defectuosa te impacta (una Excepción), la burbuja estalla, absorbe el golpe letal por completo y te permite seguir operando sin recibir daño real al núcleo del servidor.

2. Envoltorio Estándar (ApiResponse)

Para cumplir la HU-015 de enviar mensajes visuales y estandarizados, creamos la clase ApiResponse.java.

public class ApiResponse<T> {
    private String message;
    private T data;
}
  • Clase Genérica <T>: La T significa "Tipo". Esto hace que la clase sea un camaleón. Podemos usar ApiResponse<Course> para enviar un curso, o ApiResponse<List<Category>> para enviar una lista entera de categorías, o ApiResponse<Void> para no enviar nada de data (ej. al borrar algo). El contenedor sigue siendo el mismo, solo cambia lo que lleva adentro.
Analogía de Resident Evil: ApiResponse es el famoso Baúl de Objetos (Item Box). Todos los baúles del juego son idénticos por fuera y todos cumplen el propósito de resguardar tus cosas (la variable message es el baúl en sí). La letra <T> representa que adentro puedes meter una Escopeta, una Hierba, o una Llave de Picas (los datos del curso). No importa qué metas dentro, para el jugador (frontend), siempre va a interactuar con el mismo baúl estándar.

3. Validación de Protocolos URL (@URL)

@URL(protocol = "http", message = "El formato de la URL es incorrecto, debe incluir http o https")
private String courseUrl;
  • @URL de Hibernate Validator: Actúa en la puerta de entrada (el DTO). Al ponerle protocol = "http", forzamos a que obligatoriamente el usuario escriba la URL empezando con HTTP o HTTPS, evitando formatos ambigüos como "www.google" que podrían romper enlaces en el frontend.
